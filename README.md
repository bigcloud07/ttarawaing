# 따라와잉

출발 장소와 도착 장소를 고르면 아래 여정을 한 번에 이어주는 따릉이 통합 길찾기 프로토타입입니다.

`출발지 → 도보 → 대여소에서 대여 → 자전거 → 대여소에 반납 → 도보 → 목적지`

## 현재 프로토타입에서 되는 것

- 카카오맵 장소 자동완성 및 키보드 탐색
- 출발·도착 장소 바꾸기와 현재 위치 사용
- 서울자전거 운영 목록에서 가장 가까운 대여소 자동 선택
- 목적지와 가까운 반납 대여소 3곳 추천
- 공식 서울자전거 현황을 통한 대여 가능 자전거 수 반영
- 도보·자전거·도보 3개 구간의 실제 도로 경로선과 예상 시간·거리 표시
- 대체 반납 대여소 선택 시 경로 재계산
- 출발·대여·반납·도착 4개 지점을 카카오맵 자전거 길찾기로 전달
- 데스크톱 분할 화면과 모바일 지도 우선 레이아웃

대여소 위치는 서울시 2026년 6월 공식 자료와 서울자전거 운영 목록을 사용합니다. 대여 가능 자전거 수는 서울자전거 실시간 현황을 우선하며, 연결이 지연되면 2026년 7월 20일 운영 목록 스냅샷으로 추천을 유지합니다. 빈 반납 슬롯은 신뢰할 수 있는 공식 필드가 없어 표시하지 않습니다. 지도 경로선은 OpenStreetMap 도로망을 따라 그리며, 화면의 시간·거리는 추천 속도를 위한 예상치입니다.

## 실행

Node.js 22.13 이상에서 실행합니다.

```bash
npm install
npm run dev
```

배포용 빌드와 렌더링 검증:

```bash
npm test
```

## 데이터 연동

장소 검색과 지도 표시는 카카오맵 JavaScript SDK를 사용합니다. `KAKAO_JAVASCRIPT_KEY`는 런타임 환경 변수로만 제공하고 저장소에는 넣지 않습니다. 카카오맵 이어보기는 출발지, 대여소, 반납 대여소, 목적지를 한 번에 전달합니다.

도보·자전거 경로 형상은 비공개 저트래픽 프로토타입에 한해 FOSSGIS가 운영하는 OSRM 공개 라우터를 사용합니다. 세 구간은 이용 정책에 맞춰 1초 이상 간격으로 순차 조회하고 좌표쌍별 결과를 캐시합니다. 건물 중심점이 보행망의 잘못된 지점에 붙어 과도하게 우회하는 짧은 도보 구간은 직선 점선으로 보정하며, 서비스가 응답하지 않을 때도 실제 경로인 것처럼 보이지 않도록 전체 선을 `예상 연결선`으로 표시합니다. 공개·상용 출시 전에는 자체 라우터 또는 SLA가 있는 정식 라우팅 API로 교체해야 합니다.

운영 대여소와 대여 가능 자전거 수는 서울자전거 공식 사이트 현황을 브라우저에서 우선 조회하고 서버 경로를 보조로 사용합니다. 연결에 실패해도 비운영 대여소를 추천하지 않도록 최근 운영 목록 스냅샷을 앱에 포함합니다.

현재 서울자전거 현황 엔드포인트는 공식 사이트에서 쓰지만 공개 문서화된 API는 아닙니다. 제품화 단계에서는 서울 열린데이터광장 `bikeList` API 키를 발급받아 교체하는 편이 안전합니다. 공식 응답에는 신뢰할 수 있는 빈 반납 슬롯 수가 따로 없어 반납 대여소는 운영 상태와 목적지 거리를 우선합니다.
